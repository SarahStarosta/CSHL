Initialize CellBase

Copy CellBase functions to you computer. Add the directories containing your CellBase functions to the top of your Matlab path (see addpath). Call initcb to initialize your CellBase. You have to specify a main CellBase directory and a main CellBase mat file. This mat file will contain the list of cells in your CellBase (CELLIDLIST), the list of the analyses you performed (ANALYSES) and the output of your analyses (TheMatrix).

Directory Sturcture

Your main directory (e.g. mPFC Data) is the directory you specify when initializing cellbase.

Use the following naming convention for subdirectories: animal/subject name and number, e.g. 'd025' ('d' as your personal or project identifier, '025' for the animal/subject ID). Please use four-character directory names!

Session-directories: within each animal/subject subdirectory, there may be session directories for that animal/subject. Use a YYMMDDs naming convention (s is a single character; e.g. 100313a is the first session on 13th March 2010). NOTE: Even if there is only 1 session on a given day, it should still be named with the suffix 'a'!

Files within a session directory may contain various file types pertaining to that session, e.g. behavioral data, raw recording files, MClust files for clustered spikes, information files on stimulation protocols.

Multiple instances of CellBase

You can have multiple CellBases on your computer. You have to initialize all CellBases with initcb. You will be required to give a unique name to each CellBase.

You can specify your current (default) CellBase using choosecb. whichcb returns the name of your current CellBase. whoscb returns the names of all initialized CellBases. You can rename a CellBase with renamecb or change the path of a CellBase with relocatecb . You can delete a CellBase with deletecb.
