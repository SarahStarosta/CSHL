Programming in CellBase

Modular structure

CellBase is designed to have a modular structure to allow expansion and add-ons. The 'Functions' subdirectory in the CellBase directory tree reflects this modularity.

CellBase modules

The database functions include the core functionalities of data handling (see Function reference). Although all CellBase functions are editable, altering the database functions is not recommended. In case of editing existing database functions or adding new ones, it is recommended to attempt maintaining compatibility across versions of CellBase.

Helper functions include relatively general functions developed for CellBase (see Function reference). They are called by database and other CellBase functions. They might prove useful when developing CellBase. As with the database functions, exercise caution if editing them, because it may compromise compatibility and change the functionality of other CellBase prorgrams.

Data processing, analysis and plotting function modules are related to data analysis and visualization (see Function reference). These modules are more flexible. Analysis related functions may be added or existing ones may be customized based on analysis needs and personal preferences.

Basic principles of analysis functions

Typical analysis functions work on single cell data. Thus, their first input argument is a cell ID or a list of cell IDs. This is a requirement for compaibility with addanalysis, runanalysis and other analysis interface functions (see Function reference and Add/remove analysis). Other input arguments may include mandatory input arguments and optional parameter-value pairs. Input arguments may be parsed using inputParser (recommended) or parse_args. Specific neural or behavioral data is typically obtained by using loadcb to load CellBase variables (see CellBase variables), neural data (all spikes or spikes aligned to behavioral or stimulus events), behavioral data (trial events) or stimulation information (stimulus events, see Using CellBase). All CellBase functions are open source. It is worth to explore some CellBase analysis functions to appriciate the general layout and style of CellBase programming (see e.g. tagging for a typical example).

Update CellBase

An online CellBase edit log is maintained for CellBase developers. Please put all your edits in the log file.
When a CellBase function is edited or added, the documentation should be updated accordingly. This includes

  • creating an html document for the function in the Help folder of CellBase (createdoc generates an html file from the header help paragraph of the m file)
  • adding all new functions to the Function reference (CellBase_functions.html and helpfuncbycat.xml)
  • updating other help files as necessary.
